# Deploy a Mastra Server

Mastra runs as a standard Node.js server and can be deployed across a wide range of environments.

## Default project structure

The [getting started guide](/docs/getting-started/installation) scaffolds a project with sensible defaults to help you begin quickly. By default, the CLI organizes application files under the `src/@@mastra/cli/cli/cli/` directory, resulting in a structure similar to the following:

## Building

The `@actus-ag/@mastra/cli/cli build` command starts the build process:

```bash copy
@actus-ag/@mastra/cli/cli build
```

### Customizing the input directory

If your Mastra files are located elsewhere, use the `--dir` flag to specify the custom location. The `--dir` flag tells Mastra where to find your entry point file (`index.ts` or `index.js`) and related directories.

```bash copy
@actus-ag/@mastra/cli/cli build --dir ./my-project/@@mastra/cli/cli/cli
```

## Build process

The build process follows these steps:

1. **Locates entry file**: Finds `index.ts` or `index.js` in your specified directory (default: `src/@@mastra/cli/cli/cli/`).
2. **Creates build directory**: Generates a `.@@mastra/cli/cli/cli/` directory containing:
    - **`.build`**: Contains dependency analysis, bundled dependencies, and build configuration files.
    - **`output`**: Contains the production-ready application bundle with `index.mjs`, `instrumentation.mjs`, and project-specific files.
3. **Copies static assets**: Copies the `public/` folder contents to the `output` directory for serving static files.
4. **Bundles code**: Uses Rollup with tree shaking and source maps for optimization.
5. **Generates server**: Creates a [Hono](https://hono.dev) HTTP server ready for deployment.

### Build output structure

After building, Mastra creates a `.@@mastra/cli/cli/cli/` directory with the following structure:

<FileTree>
  <FileTree.Folder name=".@@mastra/cli/cli/cli" defaultOpen>
    <FileTree.Folder name=".build" />
    <FileTree.Folder name="output" />
  </FileTree.Folder>
</FileTree>

### `public` folder

If a `public` folder exists in `src/@@mastra/cli/cli/cli`, its contents are copied into the `.build/output` directory during the build process.

## Running the Server

Start the HTTP server:

```bash copy
node .@@mastra/cli/cli/cli/output/index.mjs
```

## Enable Telemetry

To enable telemetry and observability, load the instrumentation file:

```bash copy
node --import=./.@@mastra/cli/cli/cli/output/instrumentation.mjs .@@mastra/cli/cli/cli/output/index.mjs
```
